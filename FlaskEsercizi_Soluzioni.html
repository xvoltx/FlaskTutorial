ESERCIZIO 1 / BASE

Registro studenti con media

L'obiettivo è ricostruire l'app dell'Esempio 5 e aggiungere il calcolo della media dei voti. La novità rispetto all'esempio è tutta nella funzione home(): dopo aver letto gli studenti dal database, si calcola la media in Python e la si passa al template come variabile aggiuntiva.

📌 La novità rispetto all'Esempio 5

Nell'Esempio 5 render_template riceveva solo studenti=lista_studenti. Qui aggiungiamo anche media=media_voti. Il template potrà così mostrare entrambe le variabili.

pythonapp.py
from flask import Flask, render_template, request, redirect, url_for
import sqlite3

app = Flask(__name__)
# Suggerimento: usa os.path per creare il DB sempre nella stessa
# cartella di app.py, indipendentemente da dove lanci il terminale.
# import os
# DB = os.path.join(os.path.dirname(__file__), "database.db")
DB = "database.db"

def get_db():
    # "with get_db() as conn" apre la connessione al DB e la chiude
    # automaticamente alla fine del blocco, salvando le modifiche.
    conn = sqlite3.connect(DB)
    conn.row_factory = sqlite3.Row
    return conn

def init_db():
    with get_db() as conn:
        conn.execute("""
            CREATE TABLE IF NOT EXISTS studenti (
                id    INTEGER PRIMARY KEY AUTOINCREMENT,
                nome  TEXT NOT NULL,
                voto  INTEGER
            )
        """)

@app.route("/")
def home():
    with get_db() as conn:
        lista_studenti = conn.execute("SELECT * FROM studenti").fetchall()

    # Calcoliamo la media in Python prima di passarla al template.
    # Se la lista è vuota evitiamo la divisione per zero: media_voti
    # rimane None e il template la mostrerà solo se non è None.
    if lista_studenti:
        media_voti = sum(s["voto"] for s in lista_studenti) / len(lista_studenti)
        media_voti = round(media_voti, 1)  # arrotondiamo a 1 decimale
    else:
        media_voti = None

    # "studenti" e "media" sono i nomi che useremo nel file HTML.
    return render_template("index.html",
                           studenti=lista_studenti,
                           media=media_voti)

@app.route("/aggiungi", methods=["POST"])
def aggiungi():
    nome = request.form["nome"]
    voto = request.form["voto"]
    with get_db() as conn:
        conn.execute(
            "INSERT INTO studenti (nome, voto) VALUES (?, ?)",
            (nome, voto)
        )
    return redirect(url_for("home"))

@app.route("/elimina/<int:id>")
def elimina(id):
    with get_db() as conn:
        conn.execute("DELETE FROM studenti WHERE id = ?", (id,))
    return redirect(url_for("home"))

# Questo blocco viene eseguito solo quando lanci direttamente
# il file con "python app.py".
if __name__ == "__main__":
    init_db()
    app.run(debug=True)

      <td>{{ s["voto"] }}</td>

💡 Nota didattica — perché calcolare la media in Python

La media si potrebbe calcolare anche con SQL (SELECT AVG(voto) FROM studenti), ma farlo in Python con sum() e len() mostra esplicitamente il flusso: dati letti dal DB → elaborati in Python → passati all'HTML. È esattamente il concetto che si vuole far capire in questa lezione.

ESERCIZIO 2 / MEDIO

Modifica dei record

La novità è la rotta /modifica/<id>, che gestisce due richieste diverse sulla stessa URL: un GET per mostrare il form pre-compilato con i dati attuali, e un POST per salvare le modifiche con una query UPDATE. Si introduce inoltre flash() per mostrare un messaggio di conferma.

Cos'è flash()?

flash() è una funzione di Flask che salva un messaggio che verrà mostrato una sola volta nella pagina successiva. È perfetto per le conferme dopo un salvataggio: dopo il redirect alla home il template legge il messaggio con get_flashed_messages(), lo mostra, e poi sparisce automaticamente.

⚠️ Richiede secret_key

I flash messages usano i cookie del browser per trasportare il messaggio tra le pagine. Flask ha bisogno di una chiave segreta per firmare questi cookie: app.secret_key = "...". Senza questa riga Flask darà un errore appena si chiama flash().

📌 Il pattern GET + POST sulla stessa rotta

GET /modifica/3 → Flask legge i dati attuali dello studente 3 dal DB e mostra il form pre-compilato.
POST /modifica/3 → Flask legge i nuovi valori dal form, esegue UPDATE, salva il flash message e reindirizza alla home.

pythonapp.py
# NB: importiamo anche flash per i messaggi di conferma
from flask import Flask, render_template, request, redirect, url_for, flash
import sqlite3

app = Flask(__name__)
# secret_key è obbligatoria per usare flash().
# In un progetto reale usare una stringa lunga e casuale.
app.secret_key = "chiave_segreta_da_cambiare"
# Suggerimento: usa os.path per creare il DB sempre nella stessa
# cartella di app.py, indipendentemente da dove lanci il terminale.
# import os
# DB = os.path.join(os.path.dirname(__file__), "database.db")
DB = "database.db"

def get_db():
    conn = sqlite3.connect(DB)
    conn.row_factory = sqlite3.Row
    return conn

def init_db():
    with get_db() as conn:
        conn.execute("""
            CREATE TABLE IF NOT EXISTS studenti (
                id    INTEGER PRIMARY KEY AUTOINCREMENT,
                nome  TEXT NOT NULL,
                voto  INTEGER
            )
        """)

@app.route("/")
def home():
    with get_db() as conn:
        lista_studenti = conn.execute("SELECT * FROM studenti").fetchall()
    if lista_studenti:
        media_voti = round(sum(s["voto"] for s in lista_studenti) / len(lista_studenti), 1)
    else:
        media_voti = None
    return render_template("index.html", studenti=lista_studenti, media=media_voti)

@app.route("/aggiungi", methods=["POST"])
def aggiungi():
    nome = request.form["nome"]
    voto = request.form["voto"]
    with get_db() as conn:
        conn.execute("INSERT INTO studenti (nome, voto) VALUES (?, ?)", (nome, voto))
    return redirect(url_for("home"))

@app.route("/elimina/<int:id>")
def elimina(id):
    with get_db() as conn:
        conn.execute("DELETE FROM studenti WHERE id = ?", (id,))
    return redirect(url_for("home"))

# ── La rotta modifica gestisce GET e POST sulla stessa URL ─
@app.route("/modifica/<int:id>", methods=["GET", "POST"])
def modifica(id):
    if request.method == "GET":
        # GET: legge i dati attuali e li passa al form.
        # fetchone() restituisce una sola riga (non una lista).
        with get_db() as conn:
            studente = conn.execute(
                "SELECT * FROM studenti WHERE id = ?", (id,)
            ).fetchone()
        return render_template("modifica.html", studente=studente)
    else:
        # POST: legge i nuovi valori dal form e aggiorna il DB.
        nome = request.form["nome"]
        voto = request.form["voto"]
        with get_db() as conn:
            conn.execute(
                "UPDATE studenti SET nome = ?, voto = ? WHERE id = ?",
                (nome, voto, id)  # tre valori per i tre ?
            )
        # flash() salva il messaggio; compare una sola volta
        # nella pagina successiva tramite get_flashed_messages().
        flash(f"Studente '{nome}' aggiornato con successo!")
        return redirect(url_for("home"))

if __name__ == "__main__":
    init_db()
    app.run(debug=True)

💡 Nota didattica — fetchone() vs fetchall()

In home() usiamo fetchall() perché vogliamo tutti gli studenti. In modifica() usiamo fetchone() perché conosciamo già l'id e vogliamo un solo record. fetchone() restituisce una singola riga (non una lista), quindi nel template accediamo direttamente con studente["nome"], senza bisogno di un {% for %}.
