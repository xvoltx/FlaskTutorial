01 / INTRODUZIONE

Cos'è Flask e perché usarlo?

Fino ad ora hai scritto script Python che girano sul tuo computer, e pagine HTML statiche che non fanno nulla da sole. Con Flask le due cose si incontrano: Python diventa il "cervello" che risponde alle richieste del browser e gli manda pagine HTML generate al momento.

🔄 Come funziona una richiesta web
UTENTE
Browser
chiede la pagina → ← riceve l'HTML
SERVER
Flask (Python)
legge/scrive → ← dati
DATI
SQLite DB

Flask è un microframework. Un framework è una raccolta di codice già scritto che ti risparmia il lavoro di partire da zero — come un kit di montaggio invece di costruire i pezzi uno per uno. Il prefisso micro significa che Flask è volutamente piccolo e semplice: ti dà solo l'essenziale per far funzionare un sito web, senza importi decine di regole su come organizzare il progetto. Puoi aggiungere solo quello che ti serve, quando ti serve.

💡 Analogia

Pensa a Flask come al cameriere di un ristorante: riceve l'ordine del cliente (la richiesta HTTP), va in cucina (Python + SQLite), e porta indietro il piatto (la pagina HTML).

Installazione e struttura del progetto

Apri il terminale e digita:

Struttura delle cartelle

Crea questa struttura nella tua cartella di lavoro. Flask si aspetta che i file HTML siano nella cartella templates/ e che i file CSS (e JavaScript) siano nella cartella static/. Non puoi mettere il CSS dove vuoi: se non sta in static/, Flask non riuscirà a trovarlo.

mia_app/
  ├── app.py              # il tuo server Flask
  ├── database.db        # creato automaticamente all'avvio
  ├── templates/
  │   └── index.html     # la tua pagina HTML
  └── static/
      └── style.css     # il tuo foglio di stile
💡 Come collegare il CSS nell'HTML

Non usare un percorso relativo normale come href="static/style.css". Flask mette a disposizione la funzione url_for proprio per questo: scrivi href="{{ url_for('static', filename='style.css') }}" nel tag <link> del tuo HTML. Flask calcolerà il percorso corretto in automatico.

03 / ROUTING

Il tuo primo server Flask

Il concetto centrale di Flask è il route (percorso): collega un indirizzo URL a una funzione Python. Quando il browser visita quell'indirizzo, Flask esegue la funzione e ne restituisce il risultato.

Esempio 1: due rotte semplici

Ogni @app.route associa un URL a una funzione. Puoi avere quante rotte vuoi nello stesso file.

python app.py 
from flask import Flask

# Crea l'applicazione Flask
app = Flask(__name__)

# Risponde a http://127.0.0.1:5000/
@app.route("/")
def home():
    return "<h1>Benvenuto nella home!</h1>"

# Risponde a http://127.0.0.1:5000/altra-pagina
@app.route("/altra-pagina")
def altra_pagina():
    return "<p>Questa è un'altra pagina.</p>"

# Questo blocco viene eseguito solo quando lanci direttamente
# il file con "python app.py". Serve ad avviare il server.
if __name__ == "__main__":
    # debug=True: ricarica automatico quando modifichi il codice
    app.run(debug=True)
1

Avvia il server

Nel terminale, dalla cartella del progetto: python app.py — oppure esegui direttamente il file da VS Code con il tasto ▶

2

Apri il browser

Vai su http://127.0.0.1:5000/ e vedrai la home. Poi prova http://127.0.0.1:5000/altra-pagina.

⚠️ Attenzione

Il server Flask gira solo sul tuo computer: nessun altro in internet può vederlo. Il 127.0.0.1 (o localhost) è sempre "questa macchina". È perfetto per sviluppare!

Esempio 2: URL con variabile

Puoi inserire una parte variabile nell'URL scrivendola tra parentesi angolari: <nome>. Flask cattura quel pezzo dell'indirizzo e lo passa automaticamente alla funzione come parametro.

python app.py 
# <nome> nell'URL diventa un parametro della funzione
@app.route("/saluta/<nome>")
def saluta(nome):
    return f"<p>Ciao, {nome}!</p>"

Visitando /saluta/Marco il browser mostra Ciao, Marco!. Visitando /saluta/Giulia mostra Ciao, Giulia!. La stessa funzione risponde a infiniti URL diversi.

💡 A cosa serve?

Le variabili nell'URL tornano utili quando vuoi una pagina per ogni elemento del database — ad esempio /studente/3 per mostrare i dettagli dello studente con id 3. Lo vedrai nell'esercizio 2.

GET e POST: i due metodi che userai

Ogni richiesta HTTP ha un metodo che indica l'intenzione del browser. Per ora ti bastano questi due — li incontrerai nella sezione Database quando aggiungerai il form per inserire dati.

MetodoCosa faQuando si usa
GETChiede una risorsa al serverAprire una pagina, cliccare un link
POSTInvia dati al serverInviare un form, aggiungere un record

Di default, @app.route risponde solo a GET. Per accettare anche POST devi scriverlo esplicitamente: @app.route("/aggiungi", methods=["POST"]) — lo vedrai in azione nella sezione 5.

04 / TEMPLATE

Separare Python dall'HTML con render_template

Finora hai visto che una funzione Flask può restituire una stringa HTML scritta direttamente in Python. Funziona, ma diventa scomodo non appena la pagina cresce: mescolare codice Python e tag HTML nella stessa stringa è difficile da leggere e modificare.

La soluzione è usare render_template, una funzione già inclusa in Flask che devi importare esplicitamente: invece di costruire l'HTML in Python, scrivi un normale file .html nella cartella templates/ e dici a Flask di caricarlo. Puoi anche passargli delle variabili Python che verranno inserite nell'HTML.

📌 La regola

Flask cerca automaticamente i file HTML dentro la cartella templates/. Basta mettere lì i tuoi file e chiamarli con render_template("nomefile.html").

Esempio 3: passare una lista da Python a HTML

Vogliamo mostrare in una tabella HTML una lista di studenti, ognuno con nome e voto. In Python rappresentiamo ogni studente come un dizionario con due chiavi: "nome" e "voto". Le chiavi sono semplicemente le etichette con cui poi accediamo ai dati nel template — avremmo potuto chiamarle in qualunque modo, ma nome e voto rendono il codice leggibile. L'insieme di tutti gli studenti è una lista di dizionari.

python app.py 
from flask import Flask, render_template  # NB: importiamo anche render_template

app = Flask(__name__)

@app.route("/")
def home():
    # Una lista di dizionari: ogni dizionario è uno studente.
    # "nome" e "voto" sono le chiavi con cui accederemo ai dati nel template.
    lista_studenti = [
        {"nome": "Giulia", "voto": 8},
        {"nome": "Marco",  "voto": 7},
        {"nome": "Sara",   "voto": 9},
    ]
    # render_template carica index.html e gli passa i dati.
    # "studenti" (a sinistra del =) è il nome con cui il template
    # vedrà la variabile. "lista_studenti" (a destra) è la variabile
    # Python definita qui sopra.
    return render_template("index.html", studenti=lista_studenti)

if __name__ == "__main__":
    app.run(debug=True)
html + jinja2 templates/index.html 
<!DOCTYPE html>
<html lang="it">
<head><meta charset="UTF-8"><title>Studenti</title></head>
<body>
  <h1>Lista studenti</h1>
  <table>
    <tr><th>Nome</th><th>Voto</th></tr>

    <!-- per ogni studente passato da Python, crea una riga -->
    {% for s in studenti %}
    <tr>
      <td>{{ s.nome }}</td>
      <td>{{ s.voto }}</td>
    </tr>
    {% endfor %}

</html>

Dentro i file HTML nella cartella templates/ puoi usare due tipi di tag speciali:

SintassiSignificatoEsempio

💡 Vantaggio

Con render_template l'HTML rimane un file separato che puoi modificare con tutta la comodità di VS Code — con evidenziazione, autocompletamento e anteprima — senza toccare il codice Python.

05 / FORM HTML

Raccogliere dati dall'utente con i form

Un form HTML è il modo con cui una pagina web raccoglie input dall'utente — testo digitato, numeri, scelte — e li invia al server. Lo riconosci dal tag <form>. È lo strumento che userai nell'esempio 5 per aggiungere uno studente al database.

Gli attributi fondamentali di un form

AttributoA cosa serveEsempio
actionL'URL a cui inviare i dati quando si preme il bottoneaction="/aggiungi"
methodIl metodo HTTP da usare: get o postmethod="post"
nameIl nome del campo: Flask lo usa per recuperare il valore con request.form["nome"]name="nome"
💡 method="post" o method="get"?

Usa post quando invii dati che modificano qualcosa (aggiungere, eliminare, salvare). Usa get per le ricerche, dove i dati possono stare nell'URL senza problemi.

I campi più comuni

TagA cosa serve
<input type="text">Campo di testo a riga singola
<input type="number">Campo numerico (con frecce su/giù)
<input type="email">Campo email (valida il formato automaticamente)
<button type="submit">Bottone che invia il form

Esempio 4: il form per aggiungere uno studente

Questo è il form che useremo nella sezione successiva per aggiungere uno studente. Nota l'attributo name su ogni campo: è il nome con cui Flask recupererà il valore in Python con request.form["..."]. L'attributo required impedisce di inviare il form se il campo è vuoto.

html frammento di index.html 
<!-- action="/aggiungi": quando si preme il bottone, i dati -->
<!-- vengono inviati alla rotta /aggiungi di Flask           -->
<!-- method="post": i dati viaggiano nel corpo della richiesta, -->
<!-- non nell'URL                                            -->
<form action="/aggiungi" method="post">

  <!-- name="nome": Flask leggerà questo campo con request.form["nome"] -->
  <input type="text" name="nome" placeholder="Nome studente" required>

  <!-- name="voto": Flask leggerà questo campo con request.form["voto"] -->
  <input type="number" name="voto" placeholder="Voto" min="1" max="10" required>

  <!-- type="submit": questo bottone invia il form -->
  <button type="submit">Aggiungi studente</button>

</form>
📌 Il collegamento con Flask

Il valore dell'attributo name nel form HTML e la chiave usata in request.form["..."] in Python devono essere identici. Se nel form scrivi name="voto", in Python devi scrivere request.form["voto"]. Se i nomi non coincidono, Flask non troverà il valore.

⚠️ Attenzione: il form da solo non basta

Il form dell'esempio 4 è solo un frammento HTML. Se lo metti in un file e premi "Aggiungi", il browser proverà a inviare i dati alla rotta /aggiungi — ma se Flask non è in esecuzione, o se quella rotta non esiste ancora in app.py, otterrai un errore. Il form funziona solo quando c'è un server Flask attivo con la rotta /aggiungi definita. Nell'esempio 5 della sezione successiva troverai tutto il codice completo e funzionante insieme.

06 / DATABASE

Collegare Flask a SQLite

Finalmente mettiamo insieme tutto: Python, Flask, HTML e il database. In questo esempio costruiremo una piccola app per gestire un registro di studenti. L'utente potrà vedere la lista, aggiungere uno studente tramite un form, ed eliminarne uno con un click.

Cosa farà l'app

L'app avrà tre funzionalità, ognuna gestita da una rotta diversa:

1

Visualizzare la lista (GET /)

Quando l'utente apre la pagina, Flask legge tutti gli studenti dal database e li mostra in una tabella HTML.

2

Aggiungere uno studente (POST /aggiungi)

L'utente compila un form con nome e voto e preme "Aggiungi". Flask riceve i dati e li inserisce nel database con una INSERT.

3

Eliminare uno studente (GET /elimina/<id>)

Ogni riga della tabella ha un link "Elimina". Cliccandolo, Flask cancella quello studente dal database con una DELETE e ricarica la pagina.

Le nuove funzioni che userai

Rispetto agli esempi precedenti, importeremo tre funzioni nuove da Flask:

FunzioneA cosa serve
requestLegge i dati inviati dall'utente tramite un form (es. il nome e il voto digitati)
redirectDopo aver salvato i dati, rimanda il browser su un'altra pagina (es. torna alla home)
url_forRicava l'URL di una rotta dal nome della sua funzione, così non devi scrivere l'indirizzo a mano

Oltre a queste, useremo due funzioni Python che scriveremo noi: get_db() per aprire la connessione al database, e init_db() per creare la tabella se non esiste ancora. Le vedremo nel dettaglio direttamente nel codice qui sotto.

💡 Il flusso completo

Utente apre la pagina → Flask legge il DB → passa i dati a render_template → il file HTML mostra la tabella. Quando l'utente invia il form → Flask riceve i dati con request → li salva nel DB → usa redirect per tornare alla home.

Esempio 5: registro studenti con database

python app.py 
# Importiamo Flask e le funzioni che ci servono:
# - render_template: genera l'HTML da un file .html
# - request: legge i dati inviati da un form
# - redirect: manda il browser su un'altra pagina
# - url_for: ricava l'URL di una funzione dal suo nome
from flask import Flask, render_template, request, redirect, url_for
import sqlite3

# Crea l'applicazione Flask
app = Flask(__name__)

# Nome del file database: verrà creato nella stessa cartella di app.py
DB = "database.db"


# ── get_db: apre la connessione al database ───────────────
# La chiamiamo ogni volta che abbiamo bisogno di leggere o scrivere.
# row_factory = sqlite3.Row fa sì che ogni riga restituita dalla
# query si comporti come un dizionario: possiamo scrivere
# riga["nome"] invece di riga[0]
def get_db():
    conn = sqlite3.connect(DB)
    conn.row_factory = sqlite3.Row
    return conn


# ── Crea la tabella se non esiste ancora ──────────────────
# Viene chiamata una sola volta all'avvio del server (vedi in fondo).
# IF NOT EXISTS evita errori se il DB esiste già.
def init_db():
    # "with get_db() as conn" apre la connessione al DB e la chiude
    # automaticamente alla fine del blocco, salvando le modifiche.
    with get_db() as conn:
        conn.execute("""
            CREATE TABLE IF NOT EXISTS studenti (
                id    INTEGER PRIMARY KEY AUTOINCREMENT,
                nome  TEXT NOT NULL,
                voto  INTEGER
            )
        """)


# ── Rotta 1: pagina principale ────────────────────────────
# Risponde a GET / (es. http://localhost:5000/)
# Legge tutti gli studenti dal DB e li passa al template,
# che li mostrerà nella tabella HTML.
@app.route("/")
def home():
    with get_db() as conn:
        lista_studenti = conn.execute("SELECT * FROM studenti").fetchall()
    # "studenti" è il nome con cui il template vedrà la variabile,
    # "lista_studenti" è la variabile Python definita qui sopra.
    return render_template("index.html", studenti=lista_studenti)


# ── Rotta 2: aggiunge uno studente ────────────────────────
# Risponde solo a POST /aggiungi (il form nel template usa method="post").
# request.form contiene i valori dei campi del form (nome, voto).
# I ? nella query evitano le SQL injection: non concatenare mai
# stringhe direttamente nella query!
@app.route("/aggiungi", methods=["POST"])
def aggiungi():
    nome = request.form["nome"]  # legge il campo "nome" dal form
    voto = request.form["voto"]  # legge il campo "voto" dal form
    with get_db() as conn:
        conn.execute(
            "INSERT INTO studenti (nome, voto) VALUES (?, ?)",
            (nome, voto)  # i valori reali sostituiscono i ?
        )
    # Dopo l'inserimento rimanda alla home (pattern Post/Redirect/Get):
    # evita che il browser reinvii il form se si ricarica la pagina.
    return redirect(url_for("home"))


# ── Rotta 3: elimina uno studente ─────────────────────────
# <int:id> cattura il numero nell'URL (es. /elimina/3)
# e lo passa come parametro alla funzione già convertito in intero.
@app.route("/elimina/<int:id>")
def elimina(id):
    with get_db() as conn:
        conn.execute("DELETE FROM studenti WHERE id = ?", (id,))
    return redirect(url_for("home"))


# ── Avvio del server ──────────────────────────────────────
# Questo blocco viene eseguito solo quando lanci direttamente
# il file (python app.py), non quando viene importato da altri moduli.
# init_db() crea la tabella se non esiste, poi si avvia il server.
if __name__ == "__main__":
    init_db()
    app.run(debug=True)

🔒 Sicurezza: parametri con ?

Nota l'uso dei parametri posizionali ? nelle query SQL. Non concatenare mai stringhe nelle query ("WHERE id = " + id) perché espone l'app alle SQL injection. I ? proteggono automaticamente.

row_factory: righe come dizionari

Con conn.row_factory = sqlite3.Row, ogni riga restituita dalla query si comporta come un dizionario: puoi accedere ai campi con s["nome"] oppure s.nome. Senza questa riga, le righe sarebbero solo tuple anonime e dovresti usare gli indici numerici (s[1]).
